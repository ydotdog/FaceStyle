## 模块二：TrueDepth 3D面部采集

### 2.1 ARKit人脸追踪
- [ ] 初始化 ARSession + ARFaceTrackingConfiguration
- [ ] 检测设备是否支持TrueDepth（`ARFaceTrackingConfiguration.isSupported`），不支持时给出提示
- [ ] 获取 ARFaceAnchor 的 ARFaceGeometry（1,220顶点3D网格）
- [ ] 实时显示人脸追踪状态（找到脸/未找到脸）

### 2.2 拍摄引导与质量控制
- [ ] 引导用户正面朝向相机（检测 ARFaceAnchor.transform 的旋转角，yaw/pitch/roll 均 < 10°）
- [ ] 引导适当距离（25-50cm范围内）
- [ ] 提示光照条件（利用ARFrame.lightEstimate检查亮度）
- [ ] 提示用户露出额头（可选：文字提示即可）
- [ ] 满足条件后自动/手动触发采集

### 2.3 面部数据提取与存储
- [ ] 从 ARFaceGeometry.vertices 提取关键顶点坐标（3D，单位米）
- [ ] 定义关键测量点映射（见下方顶点索引表）
- [ ] 将采集数据封装为 `FaceScanData` 模型保存

**关键顶点索引（ARKit Face Mesh）**：
```
需要标定的关键点：
- 下巴最低点（chin）
- 额头最高可见点（forehead top）
- 左右颧骨最宽点（cheekbone L/R）
- 左右下颌角（jaw angle L/R）
- 左右太阳穴（temple L/R）
- 鼻尖（nose tip）
- 左右眼外眼角（outer eye corner L/R）
```
> 注：ARKit Face Mesh顶点索引需实际调试确认对应关系，建议先做一个可视化调试工具标注顶点编号。

## 模块三：面部比例计算引擎

### 3.1 核心比例指标计算
- [ ] **脸长**：额头顶点 → 下巴底点 的3D欧氏距离
- [ ] **脸宽**：左右颧骨最宽点的3D距离
- [ ] **额头宽度**：左右太阳穴的3D距离
- [ ] **下颌宽度**：左右下颌角的3D距离
- [ ] **脸长宽比**：脸长 / 脸宽
- [ ] **额头-下颌比**：额头宽度 / 下颌宽度

### 3.2 三庭比例计算
- [ ] **上庭**：发际线 → 眉心 的距离
- [ ] **中庭**：眉心 → 鼻底 的距离
- [ ] **下庭**：鼻底 → 下巴 的距离
- [ ] 计算三庭各自占比（理想值各约1/3）
- [ ] 标记偏离程度（如"上庭偏长 → 额头较高"）

### 3.3 下颌形状分析
- [ ] 提取下颌轮廓顶点序列（左下颌角 → 下巴 → 右下颌角）
- [ ] 计算下颌角角度（钝角=圆润，接近直角=方正）
- [ ] 计算下巴尖锐度（下巴区域顶点的曲率）

### 3.4 数据模型定义

```swift
struct FaceMetrics {
    // 原始测量值（米）
    let faceLength: Float
    let faceWidth: Float
    let foreheadWidth: Float
    let jawWidth: Float
    let cheekboneWidth: Float

    // 三庭
    let upperThird: Float   // 发际线→眉心
    let middleThird: Float  // 眉心→鼻底
    let lowerThird: Float   // 鼻底→下巴

    // 计算比例
    var lengthToWidthRatio: Float
    var foreheadToJawRatio: Float
    var jawAngle: Float         // 下颌角度数
    var chinSharpness: Float    // 下巴尖锐度 0-1

    // 三庭比例
    var upperThirdRatio: Float
    var middleThirdRatio: Float
    var lowerThirdRatio: Float
}
```

## 模块四：脸型分类

### 4.1 分类规则实现
- [ ] 定义7种脸型枚举：椭圆(oval) / 圆(round) / 方(square) / 心形(heart) / 长(oblong) / 菱形(diamond) / 梨形(pear)
- [ ] 基于 FaceMetrics 的规则分类器：

```
分类逻辑（简化）：
if 脸长宽比 ≈ 1.0 && 下颌圆润        → 圆脸
if 脸长宽比 ≈ 1.0 && 下颌方正        → 方脸
if 脸长宽比 > 1.4                    → 长脸
if 额头宽 > 下颌宽 && 下巴尖         → 心形脸
if 颧骨宽 > 额头宽 && 颧骨宽 > 下颌宽 → 菱形脸
if 下颌宽 > 额头宽                   → 梨形脸
else                                → 椭圆脸（默认/理想）
```

- [ ] 输出主分类 + 置信度（0-100%）
- [ ] 支持"混合脸型"输出（如"偏心形的椭圆脸"）

### 4.2 分类结果模型

```swift
struct FaceShapeResult {
    let primaryShape: FaceShape       // 主脸型
    let confidence: Float             // 置信度
    let secondaryShape: FaceShape?    // 次要倾向
    let characteristics: [String]     // 特征描述，如["额头较宽", "下巴偏尖"]
}
```

## 模块五：发型推荐引擎

### 5.1 发型数据库构建
- [ ] 定义发型数据模型：

```swift
struct Hairstyle {
    let id: String
    let name: String            // "法式刘海 + 锁骨发"
    let description: String     // 简要描述
    let suitableFaceShapes: [FaceShape]  // 适合的脸型
    let keyBenefits: [String]   // 对该脸型的好处，如"遮盖较高的额头"
    let lengthCategory: HairLength       // 短/中/长
    let styleCategory: HairStyle         // 直/卷/编发等
    let referenceImageName: String       // 参考图片资源名
    let gender: Gender                   // 男/女/中性
}
```

- [ ] 收集整理发型数据（首批目标：男性30款 + 女性50款）
- [ ] 每款发型标注适合的脸型 + 具体原因
- [ ] 配备参考图片素材

### 5.2 推荐匹配算法
- [ ] 输入：FaceShapeResult + FaceMetrics
- [ ] 第一层筛选：按脸型匹配适合的发型集合
- [ ] 第二层排序：按具体比例特征精细打分

```
打分逻辑示例：
- 用户上庭偏长（额头高）→ 有刘海的发型加分
- 用户下颌偏宽 → 能修饰下颌的发型加分（如A-line bob）
- 用户脸长宽比偏高 → 增加横向视觉宽度的发型加分
```

- [ ] 输出 Top 5 推荐，每个附带匹配理由

### 5.3 推荐结果模型

```swift
struct HairstyleRecommendation {
    let hairstyle: Hairstyle
    let matchScore: Float              // 0-100
    let reasons: [String]              // ["能很好地修饰较高的额头", "增加下半脸的柔和度"]
    let rank: Int                      // 1-5
}
```

## 技术选型汇总

| 组件 | 选择 | 理由 |
|------|------|------|
| 语言 | Swift | iOS原生，ARKit必须 |
| UI框架 | SwiftUI | 现代、声明式、开发效率高 |
| 3D面部采集 | ARKit ARFaceTracking | TrueDepth最高精度，系统原生API |
| 数据存储 | 本地JSON/SQLite | V1.0无需服务端 |
| 图表可视化 | Swift Charts（iOS 16+） | 系统原生，无需第三方库 |
| 发型图片 | 本地资源 Assets | V1.0先用本地图片 |
